# Installing and Configuring Git

Installing and configuring Git is the foundational step towards leveraging version control in your DevOps journey. A properly installed and configured Git environment is crucial for effectively tracking changes, collaborating with others, and managing your projects. This lesson will guide you through the installation process on various operating systems and delve into essential configuration settings to personalize your Git experience.

---

## Installing Git

Git is a widely used tool, and installation packages are available for most operating systems. Let's explore the installation process for **Windows**, **macOS**, and **Linux**.

---

### Installing Git on Windows

1. **Download Git for Windows**:  
   Visit the official Git for Windows website:  
   [https://git-scm.com/download/win](https://git-scm.com/download/win)

2. **Run the Installer**:  
   After download, run the executable file. The Git Setup wizard will appear.

3. **Follow the Installation Steps**:
   - **License Agreement**: Accept the GNU General Public License.
   - **Installation Location**: Choose the default location.
   - **Components**: Select components to install. Consider adding "Git LFS".
   - **Start Menu Folder**: Choose or skip creating a Start Menu folder.
   - **Default Editor**: Choose an editor (e.g., Notepad++, Nano).
   - **Adjust PATH Environment**:
     - Use Git from Git Bash only *(recommended for beginners)*.
     - Use Git from the Windows Command Prompt.
     - Use Git and optional Unix tools *(use with caution)*.
   - **HTTPS Transport Backend**: Use native Windows Secure Channel Library.
   - **Line Ending Conversions**:
     - Recommended: *Checkout Windows-style, commit Unix-style line endings*.
   - **Terminal Emulator**: Use MinTTY (default terminal of MSYS2).
   - **Extra Options**: Enable file system caching and Git Credential Manager.
   - **Install**: Click "Install".

4. **Verify Installation**:  
   Open Git Bash or Command Prompt and run:
   ```bash
   git --version
   ```

---

### Installing Git on macOS

#### Option 1: Using Homebrew (Recommended)
1. Install Homebrew:  
   [https://brew.sh](https://brew.sh)
   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```
2. Install Git:
   ```bash
   brew install git
   ```

#### Option 2: Using Git Installer
Download from:  
[https://git-scm.com/download/mac](https://git-scm.com/download/mac)

Run the `.dmg` file and follow the installation instructions.

#### Option 3: Using Xcode Command Line Tools
Open Terminal:
```bash
git --version
```
If Git is not installed, macOS will prompt you to install it.

---

### Installing Git on Linux

#### Debian/Ubuntu:
```bash
sudo apt update
sudo apt install git
```

#### Fedora/CentOS/RHEL:
```bash
sudo dnf install git
# or
sudo yum install git
```

#### Arch Linux:
```bash
sudo pacman -S git
```

**Verify Installation**:
```bash
git --version
```

## Configuring Git

After installing Git, configure it with your personal information for commit identification.

---

### Setting Your User Name and Email

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

- `--global`: Applies to all repositories.
- Example:
  ```bash
  git config --global user.name "John Doe"
  git config --global user.email "john.doe@example.com"
  ```

---

### Configuring Your Default Text Editor

```bash
git config --global core.editor "editor_command"
```

Examples:
- Nano:
  ```bash
  git config --global core.editor "nano"
  ```
- VS Code:
  ```bash
  git config --global core.editor "code --wait"
  ```
- Sublime Text:
  ```bash
  git config --global core.editor "subl -n -w"
  ```
- Notepad++ (Windows):
  ```bash
  git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -nologo"
  ```

---

### Configuring Line Endings

```bash
git config --global core.autocrlf true
```

Options:
- `true`: (Windows) Converts LF → CRLF on checkout, CRLF → LF on commit.
- `input`: (macOS/Linux) Converts CRLF → LF on commit.
- `false`: No conversion.

---

### Configuring Default Branch Name

To use `main` instead of `master` as default branch:
```bash
git config --global init.defaultBranch main
```

---

### Verifying Configuration

List all configuration settings:
```bash
git config --list
```

Check individual settings:
```bash
git config user.name
git config user.email
git config core.editor
git config core.autocrlf
git config init.defaultBranch
```

## Practical Exercises

- **Install Git**: Follow OS-specific instructions.
- **Configure User Info**:
  ```bash
  git config --global user.name "Your Name"
  git config --global user.email "your.email@example.com"
  ```
- **Set Default Editor**:
  ```bash
  git config --global core.editor "your_editor_command"
  ```
- **Configure Line Endings**:
  ```bash
  git config --global core.autocrlf true
  ```
- **Verify Configuration**:
  ```bash
  git config --list
  ```
- **Create a Repository**:
  ```bash
  mkdir myproject
  cd myproject
  git init
  ```
  Check `.git/config` to view global configs applied.
- **Try Different Editors**: Configure different editors and observe behavior.
- **Explore Line Ending Behavior**: Create files with CRLF/LF and commit. Observe Git's handling.
